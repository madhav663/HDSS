main.go
Orchestrates all benchmarks and exports

Calls: Go-native PQC (from hybrid/), cgo PQC (from wrappers/), and classical crypto (from classical/)

hybrid/
All Go-native PQC code (Kyber, Dilithium2, etc.), with benchmarks and CSV helpers

wrappers/
Each subfolder is a self-contained Go cgo wrapper + C source/header files for a specific PQC scheme (Sphincs+, Falcon, NTRU)

sphincsplus/sphincsplus.go is your Go cgo wrapper; the rest are C files from PQClean

classical/
RSA, ECDSA, Ed25519, etc.—pure Go (standard library)

Similar benchmark functions as PQC

datasets/
Real or synthetic transaction/message data for simulating blockchain/IoT

Used for more realistic sign/verify tests

results/
All benchmarking output in CSV format

Used by plotting/analysis scripts

plot/
Python (or Jupyter Notebook) scripts for generating research-quality figures and tables

README.md
Overview, setup instructions, description of benchmarks, how to add new PQC schemes, etc.
